ARGO

Autonomous Rolling Guided Object

A smart autonomous luggage-following attachment designed for airport terminals. ARGO combines BLE tracking, ultrasonic obstacle avoidance, embedded systems, motor control, and custom PCB design to create a universal robotic suitcase platform.

View Final Presentation View Design Report

Project Overview

The Problem

Travelers often struggle with carrying heavy luggage through crowded airports, especially mobility-impaired users. Existing smart suitcases force users to purchase specialized luggage.

Our Solution

ARGO is a universal attachment that converts almost any suitcase into an autonomous following system using Bluetooth Low Energy tracking and real-time obstacle avoidance.

Key Features

• BLE smartphone tracking
• Autonomous follow mode
• Alarm mode
• 180° obstacle detection
• Custom PCB integration
• Differential motor drive system

Engineering Systems

BLE Tracking

Three ESP32-S3 modules estimate user direction and distance using RSSI values from a smartphone BLE beacon. Data is transmitted through UART to the motor control system.

Obstacle Avoidance

Two HC-SR04 ultrasonic sensors mounted on a sweeping servo provide 180° forward coverage. The system dynamically maps obstacles and executes collision avoidance maneuvers.

Motor Control

A Raspberry Pi Pico generates PWM signals to dual L298N motor drivers which control a differential drive system powered by 12V DC motors.

Custom PCB

A custom PCB designed in KiCad integrated the servo interface, ultrasonic sensors, DAC audio circuitry, and obstacle avoidance systems.

Battery System

The platform uses a custom 12-cell recycled 18650 lithium-ion battery pack with a 3S BMS and DC-DC buck converter for regulated subsystem power.

Mechanical Design

The chassis uses recycled BMW seat motors, custom 3D printed gears, acrylic panels, and SolidWorks-designed mounting systems.

Challenges & Lessons Learned

PCB Amplifier Failure

The DAC-amplifier audio circuit functioned in simulation but failed during physical implementation due to schematic discrepancies and voltage interfacing issues.

Motor Driver Heating

Continuous motor operation caused thermal buildup in the L298N drivers. Additional cooling using a 40mm fan solved the issue.

System Integration Complexity

Combining BLE tracking and obstacle avoidance into a single autonomous control loop proved significantly more challenging than standalone subsystem operation.

Key Takeaway

Real-world hardware integration often differs dramatically from simulations. Iterative debugging and subsystem isolation were critical to project success.
